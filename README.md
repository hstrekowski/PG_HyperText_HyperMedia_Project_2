## XSLT Transformation: _Reading Hobby_

Project created for the **Hypertext and Hypermedia** course. This repository presents a complete example of **XSLT transformation**, converting structured XML data into a dynamic and visually styled **HTML5** web report.
The example processes the author's _reading hobby_ data‚Äîincluding reading statistics, genre preferences, and a bibliography‚Äîto generate a user-friendly frontend interface.

## üåê Transformation Logic

The project utilizes **XSLT (Extensible Stylesheet Language Transformations)** to separate data (XML) from presentation (HTML/CSS).

The transformation process involves:

- **Template Matching:** Dedicated templates for distinct XML nodes (e.g., `Podsumowanie`, `Ksiazka`, `Media`) to ensure modular code.
- **Data Extraction:** Advanced XPath queries and predicates to navigate the document tree (e.g., accessing specific IDs or calculating totals).
- **Attribute Value Templates:** Dynamic injection of image paths (`src`) and hyperlinks (`href`) directly from XML attributes into HTML tags.

---

## üß© Project Features

- **Advanced XSLT Structures:**

  - **Conditional Rendering:** Usage of `<xsl:if>` and `<xsl:choose>` to visually distinguish "Classic" books, local publishers, or specific rating tiers (star system).
  - **Dynamic Sorting & Indexing:** Generation of an alphabetical "Quick Index" of titles using `<xsl:sort>` within a loop.
  - **Calculated Statistics:** Implementation of XPath functions (`count()`, `round()`) and variables to compute real-time data, such as the average number of pages per book or the age of a publication.

- **Presentation Elements:**
  - **Numbering Systems:** Dual-mode numbering using `<xsl:number>` (Arabic for the index, Roman numerals for the main table).
  - **Formatting:** Usage of `format-number` to handle currency-style grouping and decimal precision.
  - **Multimedia Integration:** Dynamic rendering of images and active hyperlinks based on XML parameters.

## ‚ñ∂Ô∏è How to Run and Verify

### 1. Direct Browser Rendering (Client-Side XSLT)

Open the **`ksiazki.xml`** file in a web browser. The browser will read the processing instruction and apply `pierwsza.xsl` to render the view.

> **Note:** For the best experience, use **Mozilla Firefox**.
> Due to CORS (Cross-Origin Resource Sharing) security policies, Google Chrome may block local XSLT execution.

### 2. Static HTML View

Alternatively, simply open the **`pierwsza.html`** file to view the pre-generated result.
