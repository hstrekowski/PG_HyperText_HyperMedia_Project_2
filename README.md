## XSLT Transformation: _Reading Hobby_

Project created for the **Hypertext and Hypermedia** course. This repository presents a complete example of **XSLT transformation**, converting structured XML data into:

1. A dynamic and visually styled **HTML5** web report.
2. A restructured **XML** document with a new hierarchy and schema.

The example processes the author's _reading hobby_ data‚Äîincluding reading statistics, genre preferences, and a bibliography.

## üåê Transformation Logic

The project utilizes **XSLT (Extensible Stylesheet Language Transformations)** to perform two distinct types of operations:

1.  **Presentation Transformation (XML $\to$ HTML):** Separating data from presentation to create a user-friendly report.
2.  **Structural Transformation (XML $\to$ XML):** Refactoring the dataset into a completely new schema with deep nesting and modified semantics.

The process involves:

- **Template Matching:** Modular processing of nodes (e.g., `Ksiazka` converted to `Wolumin`).
- **Data Extraction:** Advanced XPath queries to navigate the document tree.
- **Dynamic Generation:** Creating nodes and attributes on-the-fly based on input data values.

## üß© Project Features

### Part 1: Visual Report (HTML)

- **Advanced XSLT Structures:**
  - **Conditional Rendering:** Usage of `<xsl:if>` and `<xsl:choose>` (e.g., distinguishing "Classic" books).
  - **Dynamic Sorting & Indexing:** Alphabetical "Quick Index" generation.
  - **Calculated Statistics:** XPath functions (`count()`, `round()`) for real-time data analysis.
- **Presentation Elements:**
  - **Numbering Systems:** Dual-mode numbering (`1.` and `I.`).
  - **Multimedia:** Dynamic image rendering and active hyperlinks.

### Part 2: Structural Refactoring (XML)

- **Schema Restructuring:**
  - **Deep Nesting:** The output XML features a new hierarchy with at least 3 levels of depth (e.g., `Raport` $\to$ `Katalog` $\to$ `Wolumin`).
  - **Semantic Renaming:** Mapping source tags to entirely new names (at least 4 unique tags and 3 unique attributes).
- **Dynamic Node Creation:**
  - **`xsl:element`:** Generating tag names dynamically based on data values (e.g., creating a tag named `<Hubert>`).
  - **`xsl:attribute`:** Generating attribute names dynamically from source content (e.g., attribute name derived from unit type).
- **Node Replication:**
  - **`xsl:copy`:** Shallow copying of specific nodes (e.g., `Narodowosc`).
  - **`xsl:copy-of`:** Deep copying of entire subtrees (e.g., `ISBN` data with attributes).
